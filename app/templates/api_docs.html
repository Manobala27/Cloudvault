Authentication

CloudVault uses API keys to authenticate requests. You can view and manage your API keys in the API Keys Management console.

Bearer Token Authentication

Your API keys carry many privileges, so be sure to keep them secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

All API requests must be made over HTTPS. Calls made over plain HTTP will fail. API requests without authentication will also fail.

Example Request
curl -X GET https://your-domain.com/api/v1/files \
  -H "Authorization: Bearer cv_your_api_key_here"

Endpoints

GET
/api/v1/files

Returns a list of all files owned by the authenticated user.

Query Parameters
ParameterTypeDescription
folder_idInteger(Optional) Filter files by a specific folder ID.
Response Example
{
  "success": true,
  "data": [
    {
      "id": 1,
      "filename": "document.pdf",
      "s3_key": "user_1/document.pdf",
      "file_size": 1048576,
      "content_type": "application/pdf",
      "created_at": "2023-10-25T14:48:00",
      "folder_id": null
    }
  ]
}
POST
/api/v1/upload

Uploads a new file directly to CloudVault and Amazon S3.

Form Data Parameters
ParameterTypeDescription
fileFile(Required) The binary file payload.
folder_idInteger(Optional) The folder ID to upload this file into.
Example Request
curl -X POST https://your-domain.com/api/v1/upload \
  -H "Authorization: Bearer cv_your_api_key_here" \
  -F "file=@/path/to/local/file.jpg" \
  -F "folder_id=2"
DELETE
/api/v1/files/<id>

Moves a file to the trash, or permanently deletes it if it is already in the trash.

GET
/api/v1/search?q=<query>

Executes an advanced search across all files and folders.

Rate Limits

To protect CloudVault's infrastructure, all API keys are globally limited to 100 requests per minute.

If you exceed this rate limit, the API will respond with a 429 Too Many Requests HTTP status code. You must wait until your minute window resets before making further requests.

Error Codes

CloudVault uses conventional HTTP response codes to indicate the success or failure of an API request.

CodeStatusMeaning
200OKRequest was successful.
400Bad RequestMissing required parameters (e.g. no file attached).
401UnauthorizedNo valid API key provided.
403ForbiddenThe API key is valid, but the account is disabled or lacks permissions.
404Not FoundThe requested resource does not exist.
429Too Many RequestsYou have exceeded your 100 req/min quota.
500Server ErrorSomething went wrong on CloudVault's end (e.g. S3 upload failure).
